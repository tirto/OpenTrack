Memo
To: Dept. Technicians; Dept. Chairs
From: Zach Wilkinson
Date: 12/12/1999
Re: Windows 2000 at the College

Windows 2000 is coming…

Windows 2000, previously know as Windows NT 5.0, will be officially released on Feb 17, 2000.  Pre-release versions are circulating now, and ECS is in the process of evaluating the software as part of developing an upgrade strategy for the College.

Previous versions of Windows NT were typically used in a workgroup or departmental setting with the intent of integrating that group into a uniform computing environment.  To a certain extent, this reflects our current usage of the software.

As expected, Windows 2000 represents a major departure from previous releases, and takes “group integration” to a whole new level.  Windows 2000 has been designed with the idea that the enterprise will be integrated into one uniform computing environment.  This affects user, computer, and network management throughout the entire organization.

Because of this single management model, multiple Windows 2000 installations within an organization can interfere with each other and cause adverse and undesirable side effects.  The reason for this conflict is the way Windows systems track each other, which is the Computer Name found on the Identification tab of the Network control panel.  With previous versions of Windows, this name had to be unique only within the workgroup or NT domain in which the system was defined.  The network admin group usually assigned a globally unique name (DNS), and this name existed independently of the Computer Name.  The technical details will be discussed in a later document, but the important point is that Windows 2000 relies on DNS for all Microsoft Networking communications.  Because of this and other reasons, we ask that everyone speak with ECS before installing Windows 2000 of any form.

We believe Windows 2000 will be a technological breakthrough toward providing improved service and support for the College, and we appreciate your help and your patience while we are developing the rollout plan.

Thanks,

Zach Wilkinson
